#+TITLE: Deployment

This file owns static build and deploy behavior. Process rules live in
[[file:../CONVENTIONS.org][CONVENTIONS.org]]. The deploy artifact is just static files
in =build/=. GitHub Pages serves those files at the root custom domain from
=static/CNAME=.

* Build

#+begin_src sh
yarn format:check
yarn check
yarn lint
yarn test
yarn build
#+end_src

The build writes the final site to =build/=. =scripts/postbuild.mjs= then copies
the rendered =build/404/index.html= over =build/404.html= so static-host 404s
show real content before JavaScript loads.

* Preview

#+begin_src sh
yarn preview
#+end_src

Use preview after a build to inspect what will actually be deployed.

* Static Adapter

=svelte.config.js= uses =@sveltejs/adapter-static=. Pages are prerendered, so
the host does not need a Node server.

The generated output includes files copied from =static/=, including =CNAME=,
=.nojekyll=, =robots.txt=, =site.webmanifest=, =humans.txt=, and
=.well-known/security.txt=.

* Base Path

Production builds can set =BASE_PATH=:

#+begin_src sh
BASE_PATH=/subdir yarn build
#+end_src

Use this only when the site lives under a subdirectory. Leave it empty for a
root domain.

* Output

- Upload =build/=.
- Keep source assets in =static/=.
- Never patch files directly inside =build/=.

* GitHub Actions

- =.github/workflows/quality.yml= runs the quality gates on pull requests and
  pushes to =main=.
- =.github/workflows/lighthouse.yml= builds, previews, and runs Lighthouse CI.
- =.github/workflows/static.yaml= builds and deploys =build/= to GitHub Pages on
  pushes to =main= or manual workflow runs.

The deploy workflow leaves =BASE_PATH= unset because =nexenne.com= is served
from the domain root.

* Before Shipping

- =yarn format:check=
- =yarn check=
- =yarn lint=
- =yarn test=
- =yarn build=
- =yarn preview=
- Check a blog post, a work page, RSS, sitemap, and 404.
