* Reference Map

The graph is hub and spoke.

- *Root hubs*: [[file:../README.org][README]], [[file:../VISION.org][VISION]], [[file:../DESIGN.org][DESIGN]],
  [[file:../AGENTS.org][AGENTS]], [[file:../CONVENTIONS.org][CONVENTIONS]],
  [[file:../STYLE_GUIDE.org][STYLE_GUIDE]], [[file:../CONTRIBUTING.org][CONTRIBUTING]].
- *Spokes*: focused docs under =docs/=.
- *Summary down, deep dive up*: hubs keep the short rule an agent must remember.
  The full explanation, examples, and edge cases live in the spoke.
- *No lateral duplication*: spokes link to each other instead of copying shared
  paragraphs.
- *One owner per concept*: the registry below names the owner.

* Guidelines For Future Additions

- Add every new doc to this file.
- Open each doc with a one-line ownership statement and pointers to related docs.
- Keep root hubs short enough to hold in working memory.
- Keep spokes focused on one concept.
- Split a topic into a cluster only when one file grows past one screen and has
  at least three independently loadable subtopics.
- Project docs are Org mode. Do not add Markdown project docs.
