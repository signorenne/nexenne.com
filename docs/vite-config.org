#+TITLE: vite.config.ts

This file owns the explanation of Vite configuration. Vite is the dev server and
bundler. SvelteKit owns most app behavior, but this file also injects
build-time version metadata used by =src/lib/data.ts=.

* Current Config

=plugins: [sveltekit()]= connects Vite to SvelteKit. The =define= block exposes:

- =__APP_VERSION__= from =PUBLIC_APP_VERSION=, =APP_VERSION=, the latest git tag,
  or =v0.0.0=.
- =__GIT_COMMIT_HASH__= from =git rev-parse --short HEAD=.
- =__GIT_COMMIT_DATE__= from the last commit date as =YYYY-MM-DD=.

These constants are declared in =src/app.d.ts= and read through
=src/lib/data.ts= for footer/status metadata.

* Why Git History Matters

CI checks out full history in the workflows so =git describe --tags --abbrev=0=
can find the latest release tag. If git metadata is unavailable, the build still
succeeds with the =v0.0.0= fallback and empty commit fields.
